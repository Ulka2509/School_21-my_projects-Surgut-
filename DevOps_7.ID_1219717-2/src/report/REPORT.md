- Nginx слушает порт 80 и маршрутизирует запросы. Запросы с путём, начинающимся на /gateway, идут на сервис, работающий на порту 8087 по адресу 192.168.56.101. Запросы с путём, начинающимся на /session, идут на сервис на порту 8089 того же хоста.
Конфигурационный файл Nginx с блоком server, который реализует проксирование (reverse proxy) запросов к двум внутренним сервисам. 
`server` Определяет виртуальный сервер Nginx, который слушает HTTP-запросы. `listen 80` сервер слушает входящие соединения на 80 порту (стандартный порт HTTP). `servername myproxy` имя сервера (домен или хост), для которого применяется этот конфиг. В данном 
`location /gateway`- все запросы, начинающиеся с `/gateway`, будут проксироваться на внутренний адрес `http://192.168.56.101:8087`.
`proxypass http://192.168.56.101:8087`- проксирование запроса на указанный адрес.

- ![07](./images/n2.png)

1. Конфигурационный файл Nginx который необходим для управления обработкой сетевых соединений. Блок events это производительность и число подключений. Worker_connections 1024— задаёт максимальное количество одновременных подключений, которые один рабочий процесс (`worker process`) может обрабатывать одновременно. При 1 рабочем процессе и 1024 соединениях можно обслуживать до 1024 клиентов одновременно, численность рабочих процессов умножается на количество подключений .

2. Файл настроен для обслуживания HTTP-запросов, ведёт логи с подробной информацией, обеспечивает сетевые оптимизации для быстродействия.
3. Подключает сторонние конфигурационные файлы для расширения функциональности (виртуальные хосты, прокси).
- `include /etc/nginx/mime.types;`— подключает файл с описанием соответствия расширений файлов и MIME-типов (например, `.html` → `text/html`), чтобы сервер мог отсылать корректный заголовок Content-Type.
- `default_type application/octet-stream;`— если MIME-тип не найден, используется данный тип по умолчанию (общий бинарный поток данных).
- `log_format main '...'`— создаёт пользовательский формат логов под именем `main`.
 В логи записывается:  
  — IP клиента (`$remote_addr`)  
  — имя пользователя, если есть (`$remote_user`)  
  — локальное время запроса (`$time_local`)  
  — строка запроса (`$request`)  
  — HTTP-статус ответа (`$status`)  
  — количество отправленных байт тела ответа (`$body_bytes_sent`)  
  — адрес страницы, с которой пришли (`$http_referer`)  
  — User-Agent клиента (`$http_user_agent`)  
  — заголовок X-Forwarded-For, если присутствует (`$http_x_forwarded_for`), часто используется для определения IP клиента за proxy.

- `access_log /var/log/nginx/access.log main;`— включает ведение журнала доступа в файл `/var/log/nginx/access.log` с использованием формата `main`.

- `sendfile on;` — включает использование системного вызова `sendfile()` для более эффективной передачи файлов (минимизация копирования данных между ядром и пользователем).

`tcp_nopush on;`— оптимизация TCP, уменьшает число пакетов при отправке файлов, минимизирует задержки.

- `tcp_nodelay on;`— отключает алгоритм Нейгла, позволяет отправлять пакеты без задержек, важно для интерактивных приложений.

- `keepalive_timeout 65;`— время (в секундах), в течение которого соединение остаётся открытым для повторных запросов (позволяет клиенту отправлять сразу несколько запросов по одному TCP-соединению).

- `types_hash_max_size 2048;`— максимальный размер хэш-таблицы для хранения MIME-типов, повышает производительность поиска типа по расширению.

- `include /etc/nginx/conf.d/*.conf;`— подключает дополнительные конфигурационные файлы из каталога `conf.d`. Используется для сегментирования настроек.

- `include /etc/nginx/sites-enabled/*;`  — подключение конфигураций виртуальных хостов (сайтов), которые активированы (обычно через ссылку из sites-available).

- `include /etc/nginx/sites-enabled/proxy.conf;`— явно подключается дополнительный конфиг `proxy.conf` из той же папки. Возможно, там описаны правила обратного прокси или маршрутизация.
